# Plan: Fix Firebase Admin Configuration for User Creation

## Problem Summary
- Both Web (`/api/users`) and Mobile app (`https://lupa-puce.vercel.app/api/users`) fail to create users
- Error: "Firebase Admin no está configurado. Verifica las variables de entorno."
- The Web app on Vercel is returning HTTP 500 when attempting user creation

## Architecture
- **Web API**: `https://lupa-puce.vercel.app/api/users` (Next.js)
- **Mobile App**: Calls the same Vercel API endpoint
- **Firebase Admin Init**: `Web/lib/firebase-admin.ts` - initializes at module load time
- **Required Env Vars**: `FIREBASE_ADMIN_CLIENT_EMAIL`, `FIREBASE_ADMIN_PRIVATE_KEY`

## Root Cause Analysis

### 1. Vercel Environment Variables Issue
The most likely cause is that Vercel does NOT have the Firebase Admin environment variables configured, OR they are misconfigured:

**In `firebase-admin.ts` (lines 9-44):**
```typescript
const clientEmail = process.env.FIREBASE_ADMIN_CLIENT_EMAIL
let privateKey = process.env.FIREBASE_ADMIN_PRIVATE_KEY || ''

// Handles escaped newlines
if (privateKey && privateKey.includes('\\n')) {
  privateKey = privateKey.split('\\n').join('\n')
}

// Validation check - must have BEGIN PRIVATE KEY header
if (clientEmail && privateKey && privateKey.includes('-----BEGIN PRIVATE KEY-----')) {
  // Initialize Firebase Admin
} else {
  console.log('Firebase Admin credentials not configured correctly')
}
```

The initialization fails silently if:
- `FIREBASE_ADMIN_CLIENT_EMAIL` is missing/empty
- `FIREBASE_ADMIN_PRIVATE_KEY` is missing/empty
- Private key doesn't contain `-----BEGIN PRIVATE KEY-----` header

### 2. Environment Variable Configuration on Vercel
Vercel handles multiline environment variables (like private keys) differently than `.env.local`:
- In Vercel Dashboard, the private key should be pasted as-is with actual line breaks
- The `\n` escaping is only needed for `.env.local` files, NOT for Vercel Dashboard

### Step 4: Potential Fix - Environment Variable Format
If the private key in Vercel is stored with `\\n` (escaped backslash-n), the replacement code in `firebase-admin.ts` might not work correctly because Vercel's env var system may already convert `\\n` to actual newlines, or may not.

**Current code handles TWO cases:**
```typescript
// Case 1: Literal \n in the string
if (privateKey.includes('\\n')) {
  privateKey = privateKey.split('\\n').join('\n')
}
```

But if Vercel stores it as actual newlines, this replacement won't happen, but the code should still work because it checks for `-----BEGIN PRIVATE KEY-----` regardless.

### Step 5: If All Else Fails - Alternative Architecture
If environment variables remain problematic, consider using Firebase Functions instead of Next.js API routes for user management.
