## Problema 2: Error en web - "Firebase Admin no está configurado"

**Causa raíz:** `firebase-admin.ts` busca un archivo JSON local (`innergy-a55ba-firebase-adminsdk-fbsvc-c3cb224d4f.json`) pero en Vercel solo tenemos variables de entorno.

**Código actual:**
```typescript
const serviceAccountPath = path.join(process.cwd(), 'innergy-a55ba-firebase-adminsdk-fbsvc-c3cb224d4f.json')
if (fs.existsSync(serviceAccountPath)) {
  // usa el archivo JSON
}
```

**Solución:** Modificar `firebase-admin.ts` para usar variables de entorno en Vercel.

---

## Archivos a modificar

### 1. `Web/lib/firebase-admin.ts`
Cambiar para usar variables de entorno como fallback:

```typescript
import { initializeApp, getApps, cert } from 'firebase-admin/app'
import { getAuth as getAdminAuth } from 'firebase-admin/auth'
import { getFirestore as getAdminFirestore } from 'firebase-admin/firestore'

let app = null

try {
  const clientEmail = process.env.FIREBASE_ADMIN_CLIENT_EMAIL
  const privateKey = process.env.FIREBASE_ADMIN_PRIVATE_KEY?.replace(/\\n/g, '\n')

  if (clientEmail && privateKey) {
    app = initializeApp({
      credential: cert({
        projectId: 'innergy-a55ba',
        clientEmail: clientEmail,
        privateKey: privateKey,
      }),
    })
    console.log('Firebase Admin initialized from environment variables')
  } else {
    console.error('Firebase Admin environment variables not set!')
  }
} catch (error: any) {
  console.error('Error initializing Firebase Admin:', error.message)
  app = null
}

export const adminAuth = app ? getAdminAuth(app) : null
export const adminDb = app ? getAdminFirestore(app) : null
export default app
```
